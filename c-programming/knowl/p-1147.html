Normally, we won’t know how many numbers are in a file before we open it, so how do we know when to stop reading from the file? It turns out, the fscanf() function actually tells us when we reach the end of the file via a return value of EOF. This is so important that I’ll say it again: It is the the return value of the fscanf() function through which we’ll know when the end of the file has been reached. It is not the value that was read. This is a super common cause for confusion so be sure to ask lots of questions and make sure you understand the difference. To sum up: Instead of using a for loop to read from the file, from now on we’ll simply keep reading until we reach the end of the file, indicated by the return value of EOF of the fscanf() function.
